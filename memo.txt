# koshi プロジェクト 作業メモ（AI引き継ぎ用）
# 最終更新: 2026-07-17

## プロジェクト概要
koshiは「ターミナルネイティブ分散型SNS」。ed25519暗号で認証するソーシャルネットワーク。
- バージョン: 2.0.2
- ライセンス: MIT
- 作者: game_ryo (ryopc)

## 現在のアーキテクチャ
```
bin/cli.js          - CLI/TUI (kbコマンド)
bin/server.js       - サーバー起動スクリプト (PostgreSQL必須)
src/                - サーバーコード (Express + WebSocket)
  api/              - REST API (auth, users, posts, dms, admin, p2p)
  ws/               - WebSocket (index.js, handlers.js)
  db/               - PostgreSQL (pool.js, migrate.js, schema.sql)
  auth/             - ed25519認証 (ed25519.js, jwt.js, utils.js)
  config/           - 設定管理 (config.js) - JSON based
  local/            - ローカルモード (SQLite + Express + P2P bridge)
  p2p/              - P2P同期 (hypercore + hyperswarm) - ★ ブラウザ不可
  nostr/            - Nostr連携
client/             - React Web UI (Vite + React 18)
web/                - ビルド済み静的ファイル
```

### 6. SQLite自己完結型サーバー (新規作成済み)
**ファイル: src/local/server-standalone.js (新規作成)**
- PostgreSQL不要、SQLiteで完全動作
- Express REST API (全エンドポイント対応)
- WebSocketリアルタイム通信用
- 自動マイグレーション (起動時にテーブル作成)
- 静的ファイル配信 (Web UI)
- JWT認証 (HMAC-SHA256、外部ライブラリ不要)
- ed25519署名検証 (tweetnacl使用)
- 使用方法: `node src/local/server-standalone.js`
- 環境変数: PORT (デフォルト3000), JWT_SECRET (自動生成)

## 重要な設計判断
- **P2P放棄**: hypercore/hyperswarmはNode.jsネイティブモジュールでブラウザ不可 → 分散サーバー方式に
- **standaloneサーバー**: SQLite使用、PostgreSQL不要、誰でもワンコマンド起動可能
- **サーバーURL管理**: config.jsonのserverUrlフィールドで保存、CLIから切り替え可能
- **API互換性**: standaloneサーバーは既存のkoshi APIと完全互換
